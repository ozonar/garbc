# Garbage Collector 3

Приложение для управления файлами в указанной директории.

## Описание

Приложение предоставляет графический интерфейс (GUI) для:
- Автоматической обработки файлов при запуске:
  - Удаление файлов с расширением `.torrent`
  - Перемещение изображений (`jpg, jpeg, png, gif, bmp, webp`) в папку `!img` (если она существует)
- Ручного управления файлами:
  - Перемещение выбранных файлов в папки, названия которых начинаются с `!`
  - Удаление выбранных файлов (в корзину, если возможно, иначе безвозвратно)

## Требования

- Go 1.22 или выше
- Зависимости:
  - [Fyne](https://fyne.io) (GUI)
  - [godotenv](https://github.com/joho/godotenv) (загрузка .env)
- Для сборки под Linux необходимы системные библиотеки для GUI (libxxf86vm-dev и другие).
- Для Windows сборка возможна без дополнительных библиотек.

## Установка и запуск

1. Клонируйте репозиторий (или скопируйте файлы).
2. Установите зависимости:
   ```bash
   go mod tidy
   ```
3. Настройте файл `.env` (пример):
   ```
   WORK_DIR=H:\Desktop\ХЛАМЬ
   ```
   Можно использовать любой путь к директории, которую нужно обрабатывать.
4. Соберите приложение:
   ```bash
   go build -o garbage-collector ./cmd/garbage-collector
   ```
5. Запустите:
   ```bash
   ./garbage-collector
   ```

## Использование

1. При запуске приложение автоматически обработает файлы в указанной директории (удалит .torrent, переместит изображения).
2. В левой части окна отобразится список файлов и папок (кроме тех, что начинаются с `!`).
3. Выберите файл щелчком мыши.
4. В правой части появятся кнопки:
   - **Удалить** — удаляет выбранный файл.
   - Кнопки с именами папок, начинающихся на `!` — перемещают выбранный файл в соответствующую папку.
5. Если директория из `.env` недоступна, программа предложит выбрать другую через диалог.

## Особенности реализации

- GUI построен на библиотеке Fyne, что обеспечивает кроссплатформенность.
- Автоматическая обработка файлов происходит при каждом запуске приложения.
- Удаление файлов реализовано через `os.Remove` (без корзины). В будущем можно добавить поддержку корзины через системные API.
- Поддерживается только одиночный выбор файла (в отличие от оригинала с множественным выбором). Это упрощение из-за ограничений Fyne List.

## Сборка под Windows

Для сборки под Windows в Linux используйте кросс-компиляцию:

```bash
GOOS=windows GOARCH=amd64 go build -o garbage-collector.exe ./cmd/garbage-collector
```

Полученный исполняемый файл можно запускать на Windows без установки дополнительных библиотек (Fyne использует встроенные API).
